# 多点温度采集系统项目设计流程

## 1. 项目定位

本项目面向理论课课程设计，设计一个基于单片机的多参数环境监测系统。系统以温湿度采集为核心，同时采集可燃气体浓度和超声波距离信息，将实时数据本地显示在 OLED 上，并通过蓝牙发送到手机；系统还要把采样数据保存到内部 Flash，支持后续独立读取历史记录。当温度超过阈值或可燃气体浓度升高时，蜂鸣器按危险程度改变报警急促程度。温度报警阈值可通过旋转编码器或手机蓝牙命令调整。

基础版使用 1 个 DHT11 温湿度传感器；如题目强调“多点温度”，可扩展为多个 DHT11 分别接入独立 GPIO，软件按通道轮询采集，Flash 记录中增加 `channel_id` 字段即可。

## 2. 需求分析

| 编号 | 功能要求 | 设计落实方式 |
| --- | --- | --- |
| 1 | DHT11 检测温湿度 | GPIO 单总线时序读取 40 bit 数据，校验和验证有效性 |
| 2 | MQ-2 检测可燃气体浓度 | ADC 采集 MQ-2 AO 电压，转换为 ADC 原始值和浓度等级 |
| 3 | 超声波检测距离 | GPIO 触发 HC-SR04，Timer 捕获 Echo 高电平宽度换算距离 |
| 4 | 三项检测数据显示在 OLED | SSD1306 OLED，I2C 刷新温湿度、气体、距离、阈值、报警状态 |
| 5 | 数据通过蓝牙发送至手机 | HC-05 串口透传，周期上传文本帧，支持手机命令查询 |
| 6 | 数据保存到内部 Flash，可独立读出历史数据 | Flash 环形日志，固定长度记录，支持按条读取、导出、清空 |
| 7 | 蜂鸣器报警，危险越高声音越急促 | Timer PWM 控制蜂鸣器，温度超限和气体超限映射为不同报警周期 |
| 8 | 编码器或蓝牙命令调整温度报警阈值 | EC11 调节阈值，蓝牙 `T+`/`T-`/`SETT=xx` 修改并保存 |

## 3. 总体设计流程

1. 明确采集对象：温湿度、可燃气体浓度、距离。
2. 完成硬件资源分配：确定 GPIO、ADC、Timer、UART、I2C、Flash 使用关系。
3. 分模块设计驱动：DHT11、MQ-2 ADC、HC-SR04、OLED、HC-05、Flash、蜂鸣器、编码器。
4. 设计数据结构：统一保存一次采样快照，便于显示、上传、报警和写 Flash。
5. 设计任务调度：定时采样、定时显示、定时上传、定时保存、实时处理命令和编码器。
6. 设计报警策略：根据超限程度计算报警等级，再控制蜂鸣器鸣叫周期。
7. 设计历史数据读出流程：手机蓝牙或串口命令读取 Flash 中的历史记录。
8. 完成联调与验收：逐个模块测试，再做系统集成测试。

## 4. 系统硬件组成

| 模块 | 作用 | 建议连接 | 注意事项 |
| --- | --- | --- | --- |
| MSP430F5529 | 主控、采集、通信、存储、报警控制 | 3.3 V 逻辑 | 所有 IO 输入不得超过 3.3 V |
| DHT11 | 温湿度采集 | DATA=P1.0 | DATA 上拉 4.7 kΩ 到 3.3 V |
| MQ-2 | 可燃气体检测 | AO=P6.0/ADC12_A0 | 模块常用 5 V 供电，AO 需确认不超过 3.3 V |
| HC-SR04 | 距离检测 | Trig=P1.2，Echo=P1.3 | Echo 为 5 V 时必须分压到 3.3 V |
| OLED SSD1306 | 本地显示 | SCL=P3.1，SDA=P3.0 | I2C 上拉到 3.3 V |
| HC-05 | 蓝牙串口 | HC-05 TX=P4.5，RX=P4.4 | RX 端建议串电阻或分压保护 |
| 蜂鸣器 | 声音报警 | PWM=P2.0 | 有源蜂鸣器可直接开关，无源蜂鸣器用 PWM |
| EC11 编码器 | 阈值调节 | A=P2.1，B=P2.2，SW=P2.3 | A/B/SW 上拉，软件消抖 |
| 内部 Flash | 历史数据存储 | 片内资源 | 擦写前关中断或做临界区保护 |

## 5. 软件模块划分

| 模块 | 主要职责 | 输出数据或接口 |
| --- | --- | --- |
| `sensor_dht11` | 产生起始信号，读取温湿度 40 bit，校验数据 | 温度、湿度、有效标志 |
| `sensor_mq2` | ADC 初始化、单次采样、均值滤波、等级换算 | ADC 值、气体等级 |
| `sensor_hcsr04` | 触发测距、捕获 Echo 脉宽、距离换算 | 距离 mm、有效标志 |
| `oled_ui` | 组织 OLED 页面并刷新 | 实时页面、历史页面、阈值页面 |
| `bt_protocol` | 蓝牙命令解析、数据上传、历史导出 | 文本协议 |
| `flash_log` | Flash 擦写、环形记录、按序号读取 | 历史记录 |
| `alarm_buzzer` | 超限判断、报警等级、PWM 控制 | 蜂鸣器开关与周期 |
| `encoder_input` | EC11 方向识别、按键短/长按 | 阈值加减、页面切换 |
| `app_scheduler` | 周期任务调度和系统状态机 | 统一任务入口 |

## 6. 核心数据结构设计

```c
typedef struct {
    uint32_t seq;              /* 递增记录序号，用于历史数据排序 */
    uint32_t uptime_s;         /* 系统运行秒数，代替实时时钟 */
    int16_t temperature_x10;   /* 温度，单位 0.1 摄氏度 */
    uint16_t humidity_x10;     /* 湿度，单位 0.1 %RH */
    uint16_t gas_adc;          /* MQ-2 ADC 原始值 */
    uint8_t gas_level;         /* 气体危险等级 0~5 */
    uint16_t distance_mm;      /* 超声波距离，单位 mm */
    int16_t temp_threshold_x10;/* 当前温度报警阈值 */
    uint8_t alarm_level;       /* 综合报警等级 0~5 */
    uint16_t crc;              /* 记录校验，防止 Flash 数据损坏 */
} EnvRecord;
```

## 7. 采集与处理流程

### 7.1 DHT11 温湿度

1. 主控拉低 DATA 至少 18 ms 发起采样。
2. 释放 DATA，等待 DHT11 响应低电平和高电平。
3. 读取 40 bit 数据：湿度整数、湿度小数、温度整数、温度小数、校验和。
4. 校验通过后更新温湿度；失败则保留上一次有效值并标记传感器异常。

### 7.2 MQ-2 可燃气体

1. ADC12 选择 P6.0 通道。
2. 连续采样 8 次或 16 次，去掉最大最小值后求平均。
3. 将 ADC 值映射为气体等级：
   - 0：安全
   - 1~2：轻微升高
   - 3~4：危险
   - 5：严重危险
4. MQ-2 需要预热，系统上电前 30 秒可显示 `MQ2 WARMUP`，报警判断可延后启用。

### 7.3 HC-SR04 距离

1. Trig 输出 10 us 高电平。
2. Timer 捕获 Echo 高电平持续时间。
3. 距离换算：

```text
distance_cm = echo_us / 58
distance_mm = echo_us * 10 / 58
```

4. 超时未收到 Echo 时，距离标记为无效。

## 8. OLED 显示设计

OLED 采用 128x64 SSD1306，显示内容分为三页：

| 页面 | 内容 |
| --- | --- |
| 实时页面 | 温度、湿度、气体等级、距离、报警状态 |
| 阈值页面 | 当前温度阈值、调节方式、保存状态 |
| 历史页面 | 最近一条 Flash 记录序号、时间、温度、气体、距离 |

实时页面示例：

```text
T:26.3C  H:58%
Gas:1250 L2
Dist: 342mm
Th:30.0  ALM:0
```

## 9. 蓝牙协议设计

蓝牙采用 ASCII 文本命令，便于手机串口助手或自制 APK 调试。

| 命令 | 方向 | 功能 | 示例返回 |
| --- | --- | --- | --- |
| `?` | 手机到设备 | 查询实时数据 | `DATA T=26.3 H=58 GAS=1250 L=2 D=342 TH=30.0 ALM=0` |
| `T+` | 手机到设备 | 温度阈值加 0.5 ℃ | `TH=30.5` |
| `T-` | 手机到设备 | 温度阈值减 0.5 ℃ | `TH=29.5` |
| `SETT=32.0` | 手机到设备 | 设置温度阈值 | `TH=32.0 SAVED` |
| `HIST?` | 手机到设备 | 查询历史记录数量 | `HIST COUNT=128 FIRST=1 LAST=128` |
| `HIST 10` | 手机到设备 | 读取第 10 条历史 | `REC 10 T=... H=... GAS=... D=...` |
| `DUMP` | 手机到设备 | 连续导出全部历史 | 多行 `REC ...` |
| `CLRLOG` | 手机到设备 | 清空历史记录 | `LOG CLEARED` |

系统也可每 2 秒主动上传一次实时数据，手机端不发命令也能看到变化。

## 10. Flash 历史存储设计

内部 Flash 采用环形日志方式保存历史数据。

1. 每条记录固定长度，包含 `magic`、版本、序号、采样数据、阈值、报警等级、CRC。
2. Flash 分成若干页，每页写满后切换到下一页。
3. 当全部页写满后，擦除最旧页继续写入。
4. 上电时扫描 Flash，找到最大序号作为最新记录。
5. 蓝牙命令 `HIST?` 查询记录范围，`HIST n` 单条读取，`DUMP` 全部导出。

独立读出历史数据的关键是：历史数据不依赖 OLED 页面，也不依赖实时采样状态，只要蓝牙或串口可用，就能按序号读取 Flash 记录。

## 11. 报警策略设计

温度和气体共同决定报警等级：

```text
temp_over = max(0, temperature - temp_threshold)
gas_over  = max(0, gas_adc - gas_warning_adc)
alarm_level = max(temp_over / 2C, gas_over / 300)
alarm_level 限幅为 0~5
```

报警等级与蜂鸣器节奏：

| 等级 | 条件示例 | 蜂鸣器表现 |
| --- | --- | --- |
| 0 | 未超限 | 关闭 |
| 1 | 轻微超限 | 每 1200 ms 响 100 ms |
| 2 | 明显超限 | 每 800 ms 响 120 ms |
| 3 | 危险 | 每 500 ms 响 150 ms |
| 4 | 高危险 | 每 250 ms 响 150 ms |
| 5 | 严重危险 | 连续急促鸣叫 |

温度越高、气体浓度越大，报警周期越短，体现“越危险越急促”。

## 12. 编码器调阈值设计

| 操作 | 功能 |
| --- | --- |
| 顺时针旋转 | 温度阈值增加 0.5 ℃ |
| 逆时针旋转 | 温度阈值减少 0.5 ℃ |
| 短按 | 切换 OLED 页面 |
| 长按 | 保存当前阈值到 Flash 参数区 |

编码器 A/B 两相使用状态机判断方向，按键采用 10~20 ms 消抖，长按时间建议 800 ms。

## 13. 系统主流程

```text
上电
  |
  +-- 初始化时钟/GPIO/ADC/Timer/UART/I2C/OLED/Flash
  |
  +-- 读取 Flash 中保存的温度阈值和历史记录指针
  |
  +-- 显示启动页面，MQ-2 进入预热
  |
  +-- 主循环
        |
        +-- 每 1 s 采集 DHT11、MQ-2、HC-SR04
        +-- 计算报警等级并更新蜂鸣器节奏
        +-- 每 500 ms 刷新 OLED
        +-- 每 2 s 蓝牙上传实时数据
        +-- 每 10 s 写入一条 Flash 历史记录
        +-- 实时处理蓝牙命令
        +-- 实时处理编码器旋转和按键
```

## 14. 关键异常处理

| 异常 | 处理方式 |
| --- | --- |
| DHT11 校验失败 | 显示 `DHT ERR`，本次数据不写入 Flash 或标记 invalid |
| MQ-2 未预热 | 显示 `MQ2 WARMUP`，暂不触发气体报警 |
| HC-SR04 超时 | 距离显示 `----`，不影响温湿度和气体报警 |
| Flash 写满 | 环形覆盖最旧页，同时更新首尾序号 |
| Flash 记录 CRC 错误 | 读取时跳过该条并返回 `CRC ERR` |
| 蓝牙命令错误 | 返回 `ERR CMD`，系统继续运行 |
| 编码器抖动 | 状态机过滤非法跳变 |

## 15. 测试与验收流程

1. 单独测试 DHT11：观察温度、湿度是否稳定变化。
2. 单独测试 MQ-2：靠近可燃气体源或模拟电压变化，ADC 值应上升。
3. 单独测试 HC-SR04：改变障碍物距离，OLED 距离值应变化。
4. 测试 OLED：实时页面能显示温湿度、气体、距离、阈值、报警状态。
5. 测试蓝牙：手机发送 `?`、`T+`、`T-`、`SETT=32.0`，设备返回正确。
6. 测试 Flash：运行一段时间后发送 `HIST?`、`HIST n`、`DUMP`，能读出历史数据。
7. 测试报警：提高温度或增大 MQ-2 值，蜂鸣器应由慢变快。
8. 测试编码器：旋转能调阈值，按键能切换页面或保存阈值。
9. 系统联调：三项数据同时采集、显示、上传、存储、报警均正常。

## 16. 设计总结

本设计采用“传感器采集层、数据处理层、显示通信层、存储报警层”分层结构。DHT11、MQ-2 和 HC-SR04 分别代表数字单总线、模拟量 ADC 和定时器测距三类典型采集方式；OLED 与蓝牙提供本地和远程两种查看方式；内部 Flash 环形日志保证历史数据可独立读出；蜂鸣器和温度阈值调节形成完整的人机交互闭环。该方案硬件连接清晰、软件模块边界明确，适合作为课程设计理论作业和后续实物实现方案。
